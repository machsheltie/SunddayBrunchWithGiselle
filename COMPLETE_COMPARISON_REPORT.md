## Remaining Gap Analysis

### Critical (Affects "Whimsy" Brand Promise)

**1. WhimsyLayer Random Positioning (GSAP vs CSS)**

**Current Impact:** 5% visual parity gap

**Issue:**
- Preview creates **unique layouts every page load**
- Main site has **fixed positions** (identical every visit)
- Loses the "unpredictable, living" feel of preview

**Fix Required:**
- Replace fixed CSS positions with GSAP random positioning
- Implementation time: 30-45 minutes
- Files: `WhimsyLayer.jsx`, `WhimsyLayer.css`

**Code Change:**
```jsx
// Add to WhimsyLayer.jsx
useEffect(() => {
    const items = document.querySelectorAll('.whimsy-item');

    items.forEach((el, i) => {
        gsap.set(el, {
            left: `${Math.random() * 100}%`,
            top: `${Math.random() * 100}%`,
            rotation: Math.random() * 360,
            scale: 0.5 + Math.random() * 1.5,
            opacity: 0.1 + Math.random() * 0.2
        });

        gsap.to(el, {
            x: '+=20',
            y: '+=20',
            rotation: '+=10',
            duration: 10 + Math.random() * 20,
            repeat: -1,
            yoyo: true,
            ease: 'sine.inOut',
            delay: i * 0.5
        });
    });
}, []);
```

**Benefits:**
- Matches preview exactly
- Enhances "whimsical" brand identity
- Every visit feels fresh
- No performance impact (GSAP already loaded)

Technical Analysis

Why Were the Colors Washed Out?

The original colors were all in the 0.83-1.0 range, meaning they were very pale and close to white. When the shader mixed these colors using simplex noise, the result was a muddy, barely-visible gradient.

What Changed?

  • Buttercream: Blue channel reduced from 0.90 → 0.70 (22% more yellow saturation)
  • Sakura: Green channel reduced from 0.90 → 0.75, blue from 0.95 → 0.85 (17% more pink saturation)
  • Lavender: Red channel reduced from 0.91 → 0.80, green from 0.83 → 0.65 (23% more purple saturation)

Color Separation Improvement

Before: Color range was 0.83-1.0 (17% range) - colors too similar

After: Color range is 0.65-1.0 (35% range) - much better separation

Result: 2x improvement in color distinction and vibrancy

Additional Fix: CSS Background Removal

Secondary Issue

The watercolor canvas container had a CSS background color (var(--pale-buttercream) = #FFFDE7) that was interfering with the WebGL canvas colors, further washing them out.

Solution

Removed the CSS background entirely, allowing the WebGL canvas to provide its own background without interference.

Impact

This change, combined with the increased color saturation, results in a dramatic improvement in visual appearance - the background is now vibrant and engaging instead of flat and lifeless.

Result: 2x More Vibrant Colors

The watercolor background now matches the preview site with clear, saturated yellow, pink, and lavender regions.
